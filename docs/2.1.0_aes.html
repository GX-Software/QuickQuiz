加密算法

标准题库文件的默认加密算法为CBC模式的128位密钥AES算法，如果题库内容被压缩，则在压缩之后再进行加密。
密码规则
密码最大支持16字节字符共128位，允许使用任何ASCII编码的半角字符；当密码数量不够16字节时，重复密码直到足够16字节。
在补全密码时，末尾“\0”不参与补全，补全后的16字节密码也不包含末尾“\0”。
128位补全
AES加密算法要求加密内容长度为16字节（128位）的整倍数，在进行128位补全时，可任意选择补全的内容，不会影响最终读取。本应用程序用0xcd进行补全。
S盒与逆S盒
使用AES加密算法的标准S盒及逆S盒，不再列出。
初始向量
CBC模式所需要的初始向量如下： 
byte init_array[] = {
    0xa5, 0xa5, 0xa5, 0xa5,
    0xc3, 0xc3, 0xc3, 0xc3,
    0x5a, 0x5a, 0x5a, 0x5a,
    0x3c, 0x3c, 0x3c, 0x3c
};
兼容性测试
采用如下密钥： 
byte key[] = {
    0x2b, 0x7e, 0x15, 0x16,
    0x28, 0xae, 0xd2, 0xa6,
    0xab, 0xf7, 0x15, 0x88,
    0x09, 0xcf, 0x4f, 0x3c
};
对如下数据进行加密： 
byte in[] = {
    0x32, 0x88, 0x31, 0xe0,
    0x43, 0x5a, 0x31, 0x37,
    0xf6, 0x30, 0x98, 0x07,
    0xa8, 0x8d, 0xa2, 0x34
};
若能得出如下结果： 
byte out[] = {
    0xc6, 0xea, 0xa0, 0x35,
    0x9d, 0xdc, 0xbe, 0x5d,
    0x1b, 0xad, 0x65, 0xfa,
    0xb5, 0x55, 0xe7, 0x0e
};
且解密后可以还原数据，则证明加密算法与本应用程序完全兼容。
